## 一、需求分析

### 1.1 业务场景

现场作业人员佩戴 Rokid Glasses 智能眼镜，进行以下场景的远程协作：

- **设备巡检**：现场人员第一视角查看设备，后端专家实时指导
- **故障排查**：后端专家看到现场画面，语音指导操作步骤
- **安全监督**：远程监控现场作业是否符合安全规范
- **培训教学**：专家通过第一视角进行实操教学

### 1.2 功能需求

| 序号 | 功能 | 描述 | 优先级 |
|------|------|------|--------|
| 1 | 实时视频传输 | 眼镜端摄像头画面实时传输到后端 | P0 |
| 2 | 双向音频通话 | 眼镜端与后端双向语音通信 | P0 |
| 3 | 呼叫建立/挂断 | 信令控制呼叫生命周期 | P0 |
| 4 | 网络自适应 | 根据网络状况调整码率 | P1 |
| 5 | 本地录制 | 现场视频本地存储 | P1 |
| 6 | 截图功能 | 后端可请求眼镜端截图 | P2 |
| 7 | 电量监控 | 实时监控眼镜电量 | P2 |

### 1.3 性能指标

| 指标 | 目标值 | 说明 |
|------|--------|------|
| 视频延迟 | < 500ms | 端到端延迟 |
| 音频延迟 | < 200ms | 端到端延迟 |
| 视频分辨率 | 720p / 1080p | 可配置 |
| 视频帧率 | 15-30 fps | 自适应 |
| 音频采样率 | 16kHz / 48kHz | 宽频音频 |
| 连接建立时间 | < 3s | 呼叫接通时间 |
| 续航时间 | > 2 小时 | 连续通话 |

### 2.2 技术选型

| 组件 | 技术选型 | 说明 |
|------|----------|------|
| **眼镜端 SDK** | Rokid CXR-S SDK v1.0.9 | 眼镜端应用开发 |

## 三、详细设计

### 3.1 通信协议设计

#### 3.1.1 WebSocket 信令协议

```typescript
// 信令消息类型
enum MessageType {
  // 呼叫控制
  CALL_INITIATE = 'call.initiate',      // 发起呼叫
  CALL_ACCEPT = 'call.accept',          // 接受呼叫
  CALL_REJECT = 'call.reject',          // 拒绝呼叫
  CALL_END = 'call.end',                // 结束呼叫
  
  // WebRTC 信令
  SDP_OFFER = 'webrtc.offer',           // SDP Offer
  SDP_ANSWER = 'webrtc.answer',         // SDP Answer
  ICE_CANDIDATE = 'webrtc.ice',         // ICE Candidate
  
  // 设备控制
  DEVICE_STATUS = 'device.status',      // 设备状态上报
  DEVICE_CONTROL = 'device.control',    // 设备控制指令
  
  // 房间管理
  ROOM_JOIN = 'room.join',              // 加入房间
  ROOM_LEAVE = 'room.leave',            // 离开房间
}

// 消息结构
interface SignalingMessage {
  type: MessageType;
  roomId: string;
  from: string;
  to?: string;
  data: any;
  timestamp: number;
}
```

#### 3.1.2 呼叫流程

```
Android 手机                          信令服务器                          后端管理台
     │                                   │                                   │
     │────── room.join ─────────────────>│                                   │
     │                                   │                                   │
     │                                   │<───── room.join ──────────────────│
     │                                   │                                   │
     │<────── call.initiate ─────────────│────── call.initiate ─────────────>│
     │                                   │                                   │
     │────── webrtc.offer ──────────────>│                                   │
     │                                   │────── webrtc.offer ─────────────>│
     │                                   │                                   │
     │                                   │<───── webrtc.answer ──────────────│
     │<───── webrtc.answer ──────────────│                                   │
     │                                   │                                   │
     │<══════════════ WebRTC P2P 连接建立 ═══════════════════════════════════>│
     │                                   │                                   │
```

### 3.2 接口设计

#### 3.2.1 REST API

```yaml
# 用户认证
POST /api/v1/auth/login
  body: { username, password }
  response: { token, user }

# 呼叫管理
POST /api/v1/calls
  body: { targetDeviceId, type: 'video' | 'audio' }
  response: { callId, roomId }

GET /api/v1/calls/:callId
  response: { callId, status, startTime, duration }

POST /api/v1/calls/:callId/end
  response: { success: true }

# 设备管理
GET /api/v1/devices
  response: { devices: [{ deviceId, status, battery, lastSeen }] }

GET /api/v1/devices/:deviceId
  response: { deviceId, status, battery, firmware }

# 录制管理
GET /api/v1/recordings
  query: { deviceId, startDate, endDate }
  response: { recordings: [{ id, url, duration, createdAt }] }
```

#### 3.2.2 设备状态上报

```typescript
interface DeviceStatus {
  deviceId: string;
  online: boolean;
  battery: number;          // 0-100
  signal: number;           // -100 to -50 dBm
  temperature: number;      // 摄氏度
  firmware: string;
  lastSeen: number;         // 时间戳
}
```

### 3.3 安全设计

#### 3.3.1 认证授权

- JWT Token 认证，有效期 24 小时
- Refresh Token 机制
- 设备绑定机制（SN 码管控）

#### 3.3.2 传输加密

- WebSocket: WSS (TLS 1.3)
- WebRTC: DTLS + SRTP
- REST API: HTTPS

#### 3.3.3 数据安全

- 录制文件加密存储
- 访问日志审计
- 敏感数据脱敏
