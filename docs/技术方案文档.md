# Rokid Glasses 远程视频连线系统 - 技术方案文档

## 📋 项目概述

### 项目背景
在现场作业场景下，巡检人员佩戴 Rokid Glasses，后端专家可通过视频连线实时查看现场情况，并提供远程指导。

### 核心需求
1. **视频流**：后端可调取眼镜摄像头实时画面
2. **双向音频**：后端能听到现场声音，眼镜端能听到后端语音
3. **低延迟**：工业场景要求实时响应（<500ms）
4. **稳定性**：适应现场复杂网络环境

## 📡 通信协议设计

### 1. WebSocket 信令协议

#### 连接建立
```json
// 客户端 → 服务端：发起呼叫
{
  "type": "call_start",
  "data": {
    "device_id": "ROKID_001",
    "operator_id": "user_123",
    "timestamp": 1709294400000
  }
}

// 服务端 → 客户端：呼叫接受
{
  "type": "call_accepted",
  "data": {
    "session_id": "session_abc123",
    "webrtc_offer": { ... }
  }
}
```

#### 控制指令
```json
// 后端 → 眼镜：切换摄像头
{
  "type": "camera_switch",
  "data": {
    "camera": "front" // 或 "back"
  }
}

// 后端 → 眼镜：调节音量
{
  "type": "volume_control",
  "data": {
    "level": 0.8 // 0.0 - 1.0
  }
}

// 后端 → 眼镜：拍照
{
  "type": "take_photo",
  "data": {}
}

// 眼镜 → 后端：电量状态
{
  "type": "battery_status",
  "data": {
    "level": 75,
    "charging": false
  }
}
```

#### 通话结束
```json
// 任意一方 → 对方：结束通话
{
  "type": "call_end",
  "data": {
    "reason": "user_hangup" // 或 "network_error", "timeout"
  }
}
```

### 2. WebRTC 媒体流

#### SDP Offer/Answer 流程
```
Android 端                        后端服务器
    |                                |
    |--- createOffer() ------------->|
    |                                |
    |--- setLocalDescription() ----->|
    |                                |
    |--- WebSocket (SDP offer) ----->|
    |                                |
    |                                |--- createAnswer()
    |                                |--- setLocalDescription()
    |                                |
    |<-- WebSocket (SDP answer) -----|
    |                                |
    |--- setRemoteDescription() ---->|
    |<-- setRemoteDescription() -----|
    |                                |
    |===== WebRTC 媒体流建立 ==========|
```

#### ICE Candidate 交换
```json
// 通过 WebSocket 交换 ICE 候选
{
  "type": "ice_candidate",
  "data": {
    "candidate": "candidate:1234567890 1 udp 2122260223 ...",
    "sdpMid": "0",
    "sdpMLineIndex": 0
  }
}
```

## 🖥️ 后端服务实现

### 服务模块划分

```
backend-server/
├── app/
│   ├── main.py              # FastAPI 入口
│   ├── signaling/           # 信令服务
│   │   ├── websocket.py     # WebSocket 处理器
│   │   └── protocol.py      # 协议定义
│   ├── webrtc/              # 流媒体服务
│   │   ├── peer.py          # WebRTC Peer 连接
│   │   └── media.py         # 媒体流处理
│   ├── api/                 # REST API
│   │   ├── sessions.py      # 会话管理
│   │   └── recordings.py    # 录像管理
│   └── models/              # 数据模型
│       └── session.py       # 会话模型
├── static/                  # 前端静态文件
├── templates/               # HTML 模板
└── requirements.txt         # Python 依赖
```

### 核心接口

#### WebSocket 端点
```python
@app.websocket("/ws/signaling/{session_id}")
async def signaling_endpoint(websocket: WebSocket, session_id: str):
    await websocket.accept()
    # 处理信令消息...
```

#### REST API
```python
# 创建会话
POST /api/sessions
Body: { "operator_id": "user_123", "device_id": "ROKID_001" }

# 结束会话
POST /api/sessions/{session_id}/end

# 获取历史会话
GET /api/sessions?operator_id=user_123&limit=10
```

## 🔒 安全设计

### 认证授权
- JWT Token 认证
- 设备白名单机制
